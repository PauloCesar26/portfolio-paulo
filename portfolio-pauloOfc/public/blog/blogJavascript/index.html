JavaScript: Explorando os conceitos do DOM

Paulo Cesar
11/04/2025
Blog

De início, é preciso lembrar que o javascript é uma linguagem versátil, capaz de atuar tanto no lado do servidor (back-end) quanto no lado do cliente (front-end), que é a parte visual da aplicação. No front-end, o js interage com o DOM (Document Object Model) que representa a estrutura da página HTML e por meio dele é possível modificar os elementos modificando cores, textos, adicionando e removendo elementos e entre outras coisas que será abordado mais adiante.


Antes de tudo, é necessário entender como que javascript consegue interagir com o DOM, que lembrando é uma árvore do arquivo HTML, como assim? O html possui tags como div, h1, p, button e entre outras, elas são representadas como um nó ou objeto que o javascript acessa para fazer a manipulação.


A partir disso, para ele chegar a esse conteúdo é usado o “document” que vai selecionar o elemento que queremos, para fazer com que ele funcione e faça o que desejamos tem que adicionar outras propriedades que vão apontar o elemento certo que vai sofrer alguma ação ou alteração. Para isso, existe os métodos, são eles: getElementById que pega o id de um item no html e traz para o arquivo javascript, getElementByClassName mesma função só que ele pega uma classe (class), querySelector acessa qualquer item que corresponde ao seletor do CSS, por exemplo, se for um ponto dentro parênteses que corresponde a uma classe ele vai pegar a classe correspondente, já o querySelectorAll vai selecionar todos os elementos que corresponde ao que está dentro do parênteses, como se fosse uma lista de todos que possui a mesma identificação.
